# 飞猪房型数据 - 不用抓包的两种做法

不抓包、不碰接口，只靠 Appium/手机端拿数据，可以用下面两种方式。

---

## 方案一：WebView 取 HTML 再解析（推荐）

若房型列表在 **WebView** 里，用 Appium 切到 WebView 的 context，拿到的 Page Source 就是 **HTML**，再用脚本从 HTML 里解析房型名、价格。

### 步骤

1. **在 Appium 里看当前有哪些 context**
   - 代码：`driver.contexts` 或 Inspector 里看 “Context” 下拉。
   - 常见：`NATIVE_APP`、`WEBVIEW_com.taobao.trip` 或 `CHROMIUM`。

2. **若有 WEBVIEW_xxx**
   - 切过去：`driver.switch_to.context('WEBVIEW_com.taobao.trip')`（名称以实际为准）。
   - 再执行 **Get Page Source**，把结果保存为 `xml/feizhu_webview.html`（或任意 .html 文件）。

3. **用脚本解析 HTML**
   ```bash
   python Feizhu/parse_feizhu_html.py xml/feizhu_webview.html
   ```
   - 会尽量从 HTML 里识别房型名、价格、备注，生成 `Feizhu/1.json`。
   - 若飞猪 H5 的 class/id 和脚本里写的不一致，把一份 **feizhu_webview.html 片段**（含一两个房型卡）发出来，可针对性改解析规则。

**前提**：房型页确实是 WebView，且 App 允许切到该 context（部分 App 会禁用或只有原生）。

---

## 方案二：截图 + OCR

完全不依赖 UI 树和网络：对房型列表页**截图**，用 **OCR** 识别图中的文字，再从识别结果里用正则/规则解析出房型名、价格。

### 步骤

1. **截图**
   - Appium：`driver.save_screenshot('room_list.png')`
   - 或 uiautomator2：`d.screenshot('room_list.png')`
   - 确保截的是「房型列表」那一屏，房型名、价格在画面里。

2. **OCR**
   - 本地：用 **PaddleOCR**、**Tesseract** 等对 `room_list.png` 做文字识别，得到一段文本（或带坐标的文本块）。
   - 或把截图丢给带 OCR 的云服务/API，拿到文本。

3. **从文本里解析**
   - 用正则匹配价格（如 `¥\d+`、`\d+元`）、房型名（通常是一行或几行连续中文）。
   - 可写一个小脚本：读 OCR 结果 txt → 正则提取 → 输出与 1.json 同结构的 JSON。

**缺点**：布局一变或字体/分辨率不同，容易漏或错；多屏要多次截图、拼接。适合作为“实在没有 WebView 也没有抓包”时的备选。

---

## 对比

| 方式           | 条件               | 稳定性 | 实现难度 |
|----------------|--------------------|--------|----------|
| WebView 取 HTML | 房型页在 WebView 且能切 context | 高     | 中（需一份 HTML 样本调解析） |
| 截图 + OCR     | 无                 | 较低   | 中（正则/规则要随页面调）     |

**建议**：先按方案一试 WebView 取 HTML；若没有 WebView 或取不到，再用方案二截图 OCR。两种都不需要抓包。
