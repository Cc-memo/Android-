# 飞猪房型数据 - 抓包接口方案（手机端照样能做）

UI 树里没有房型文案时，**不用依赖界面元素**，改用 **抓包拿接口数据**，一样能拿到房型列表并生成和现在一样的 `1.json`。

---

## 一、思路

1. 手机和电脑在同一 WiFi，或电脑开热点给手机连。
2. 电脑上装抓包工具，手机设置代理指向电脑。
3. 打开飞猪 App，进入某个酒店的**房型列表页**，让接口请求发出来。
4. 在抓包工具里找到**房型/价格列表**对应的接口（一般是 JSON）。
5. 把该接口的**响应 body** 保存成文件（如 `feizhu_api_response.json`）。
6. 用项目里的脚本把接口 JSON 转成 `Feizhu/1.json`（和 UI 解析出来的格式一致）。

这样：**手机端照样操作 App 到房型页**，只是数据来源从「UI 树」换成「接口」，手机端完全做得下去。

---

## 二、抓包步骤（简要）

### 1. 电脑端

- 安装 **Charles**、**Fiddler** 或 **mitmproxy** 任一即可。
- 启动抓包，记下电脑本机 IP（如 192.168.1.100）和代理端口（如 8888）。

### 2. 手机端

-  WiFi 设置里给当前网络设**代理**：主机 = 电脑 IP，端口 = 8888。
- 若用 HTTPS：Charles/Fiddler 需要装证书到手机，按各自说明操作（Charles 需在手机浏览器访问 chls.pro/ssl 下载证书并安装）。

### 3. 抓接口

- 打开飞猪，进入一家酒店的**房型列表页**（有价格、房型名的那一屏）。
- 在电脑上的抓包工具里看请求列表，找 **域名像飞猪/淘宝** 的（如 `*.taobao.com`、`*.fliggy.com`、`*.alibaba.com` 等）。
- 点开某个请求，看 **Response** 是否是 JSON，且里面是否有 **房型名、价格、早餐、可订状态** 等字段。
- 找到后：把该请求的 **完整响应 body** 复制出来，保存为 `xml/feizhu_api_response.json`（或任意路径，后面脚本用）。

---

## 二（补充）Fiddler 具体操作（你已打开 Fiddler 时按这个做）

### 步骤 1：让 Fiddler 能抓本机 + 手机

1. 菜单 **Tools → Options → Connections**：
   - 勾选 **Allow remote computers to connect**，端口一般是 **8888**（记下）。
   - 若改过端口，以你当前为准。
2. 菜单 **Tools → Options → HTTPS**：
   - 勾选 **Decrypt HTTPS traffic**（解密 HTTPS），这样才能看到飞猪接口内容。
   - 弹窗提示安装证书时选“是”。
3. 关闭 Fiddler 再重新打开，使设置生效。

### 步骤 2：查电脑 IP

- 命令行执行 `ipconfig`，看当前连的 WiFi 的 **IPv4 地址**（如 192.168.1.100）。手机代理要填这个。

### 步骤 3：手机连代理 + 装证书（抓 HTTPS 必须）

1. 手机连**和电脑同一个 WiFi**（或电脑开热点给手机连）。
2. 进入该 WiFi 的**代理设置**：
   - **主机名**填电脑 IP（如 192.168.1.100）；
   - **端口**填 8888。
3. 在手机**浏览器**里访问：`http://电脑IP:8888`（如 `http://192.168.1.100:8888`）。
4. 页面里点 **FiddlerRoot certificate** 或类似链接，下载并安装证书（按提示“允许安装来自该来源”等）。**Android** 可能需在 设置→安全→加密与凭据 里选“从存储设备安装”选刚下的证书。

### 步骤 4：抓飞猪房型接口

1. 确认 Fiddler 左下角是 **Capturing**（你截图里已是）。
2. 在**手机**上打开飞猪 App，进入一家酒店的**房型列表页**（能看到多间房型、价格的那一屏），可下拉刷新一下。
3. 回到 **Fiddler**，左侧会话列表会多出很多行。在 **Host** 列找飞猪/阿里系域名，例如：
   - `*.taobao.com`、`*.fliggy.com`、`*.alibaba.com`、`*.alicdn.com` 等。
4. 逐个点选这些请求，右侧切到 **Inspectors → JSON** 或 **Raw**，看 **Response** 里是否有 **房型名、价格、列表** 等。
5. 找到**房型列表数据**所在的那条请求后：
   - 右侧 **Response**  body 全选复制，或 菜单 **File → Save → Response Body** 存为 `xml/feizhu_api_response.json`。

### 步骤 5：转成 1.json

在项目根目录执行：

```bash
python Feizhu/feizhu_api_to_json.py xml/feizhu_api_response.json
```

会生成 `Feizhu/1.json`。若接口字段和脚本不匹配，把该接口的响应样本（可打码）发出来可再改解析。

**抓完后**：记得把手机 WiFi 的代理关掉，否则不上网时也会走 Fiddler。

## 三、怎么快速找到「房型列表」那条请求（请求太多时）

列表里很多请求时，按下面顺序筛，能很快缩小范围：

### 1. 先看 Host（域名）——最有用

- **优先看**：Host 里带 **taobao**、**fliggy**、**alitrip**、**alibaba** 的（飞猪/阿里系）。
- **可先忽略**：
  - `zijieapi.com`、`byteimg.com` 等（字节跳动，多半是统计/监控）；
  - `img.alicdn.com`（一般是图片 CDN，不是接口）；
  - `baidu.com`、`cup.com.cn` 等（地图、支付等，不是房型列表）。

在 Fiddler 里可以点 **Filters**，在 Host 里填 `taobao` 或 `fliggy`，只显示这些域名的请求。

### 2. 看 Result（状态码）

- **优先点开**：**200**（成功，有返回内容）；
- 204 多为“无内容”，一般不是房型数据；
- **429、502** 等是错误，可跳过。

### 3. 看 URL 路径（辅助）

房型/价格接口的 URL 里常带类似：`room`、`rate`、`product`、`detail`、`list`、`hotel`、`item` 等。  
像 `monitor_pc/collect`、`/collect/ap` 这类多半是埋点上报，不是房型接口。

### 4. 最终确认：看 Response 内容

点中某条请求后，右侧 **Inspectors → JSON** 或 **Raw** 看 **Response**：

- 若是 **JSON**，且里面有 **房型名、价格、列表**（如 `roomName`、`price`、`list`、`rates` 等），就是你要的那条；
- 若是 HTML、或一堆乱码、或只有 `{"code":0}` 没有列表，就换下一条。

### 5. 操作顺序建议

1. 用 **Filters** 把 Host 限制为 `taobao` 或 `fliggy`；
2. 在剩下列表里专挑 **Result=200**、URL 里带 `room`/`rate`/`hotel`/`list`/`detail` 的；
3. 逐个点开看 Response，找到第一条“JSON + 房型列表”的，保存其 Response Body 为 `xml/feizhu_api_response.json` 即可。

## 四、把接口 JSON 转成 1.json

项目里已提供脚本 **Feizhu/feizhu_api_to_json.py**，用于把接口返回的 JSON 转成和 UI 解析一致的 `1.json` 格式。

### 用法

```bash
# 指定接口响应文件（你抓包保存的 JSON）
python Feizhu/feizhu_api_to_json.py xml/feizhu_api_response.json

# 输出默认写到 Feizhu/1.json；也可指定输出路径
python Feizhu/feizhu_api_to_json.py xml/feizhu_api_response.json -o Feizhu/1.json
```

### 接口结构不确定时

飞猪接口字段名可能叫 `roomName`、`name`、`price`、`amount`、`breakfast` 等，每家接口不一。  
脚本里做了常见字段的自动映射；若你抓到的 JSON 结构不同，把 **一份真实的接口响应样本**（可打码敏感信息）发出来，我可以按实际字段帮你改 **feizhu_api_to_json.py** 的解析逻辑。

## 五、抓包失败时（只有 Tunnel、看不到飞猪接口）

若 Fiddler 里**一直只有「Tunnel to …:443」**、没有飞猪域名（taobao/fliggy），或点开请求也看不到 JSON 响应，通常有两种情况。

### 情况 A：还没抓到飞猪的请求

- 抓包时**必须**在手机上打开飞猪，并进入**酒店详情 → 房型列表页**（能看到多间房型、价格的那一屏），必要时下拉刷新。
- 若列表里只有 `cup.com.cn`、`xiaomi.com`、`baidu.com` 等，说明当前没有飞猪的请求，先按上面操作再清空会话重新抓一次。

### 情况 B：飞猪不信任你装的 CA（解密失败）

- 你装的 Fiddler 证书是「用户 CA」；**Android 7+** 下，很多 App（含飞猪）默认**不信任用户 CA**，只信系统 CA，所以会一直走隧道、不解密，你只能看到 Tunnel。
- 飞猪还可能做了**证书绑定（Certificate Pinning）**，即使证书在系统里，也可能只连自己的服务器证书，照样不解密。

**可行做法：**

1. **用模拟器 + 系统证书**（推荐尝试）  
   在 **夜神 / MuMu / 雷电** 等模拟器里，将 Fiddler 根证书导入为**系统证书**（需 root 或模拟器自带“安装到系统”），再在模拟器里装飞猪、设代理，往往能解密。
2. **接受抓包不可行，改用截图 OCR**  
   对房型列表页截图，用 PaddleOCR 等识别文字，再从识别结果里解析房型名、价格，见 **Feizhu/不用抓包方案.md** 的「方案二」。

---

## 六、小结

| 方式           | 是否还能做 |
|----------------|------------|
| UI 树解析      | 飞猪当前页面无文案节点，做不了。 |
| **抓包 + 接口 JSON** | **可以**：手机照常点到房型页，用接口数据生成 1.json。 |
| WebView HTML   | 若能切到 WEBVIEW 并拿到 HTML，也可以写解析，需样本。 |

所以：**手机端完全可以做，改成「抓包拿接口 → 用脚本转 1.json」即可。**
